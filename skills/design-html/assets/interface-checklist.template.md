# Interface Checklist

- Scope source and non-goals are explicit.
- Primary journey is visible.
- Information hierarchy is clear.
- Loading state is defined.
- Empty state is defined.
- Validation state is defined.
- Error / recovery state is defined.
- Unauthorized / permission state is defined where relevant.
- Success / confirmation state is defined.
- Responsive behavior is named.
- Accessibility notes are named.
- HTML/CSS delivery constraints are named.
