# Product Intent to UX

Convert product scope to interface work in this order:

1. Scope source
   - Quote the approved source or user brief in one sentence.
   - Record non-goals so the interface does not grow the product.
2. User job
   - Identify who is acting, what they are trying to accomplish, and what blocks them today.
3. Journey
   - Map the main path, the key fork/decision points, and the recovery path when things fail.
4. Screen inventory
   - Name the minimum screens, panels, or states required to complete the job.
5. Interface priorities
   - Decide what must be visible first, what can be progressive disclosure, and what must never be hidden.
6. State coverage
   - Cover loading, empty, validation, error, unauthorized, and success behavior.

Do not introduce a new product objective just because it would make the UI more elegant.
