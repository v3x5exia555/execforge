# Design HTML Quality Bar

Reject vague output. A good `design-html` result is:

- Product-faithful: no silent scope expansion
- User-centered: the main job is obvious in the first screen structure
- State-complete: loading, empty, error, and success states are all explicit
- Accessible: semantic structure and interaction basics are named, not implied
- Implementable: HTML/CSS guidance is specific enough to build from

Warning signs:

- Mostly visual adjectives with no interaction logic
- No empty or failure states
- No distinction between primary and secondary actions
- Responsive behavior reduced to "make it mobile friendly"
- Accessibility reduced to "follow best practices"
