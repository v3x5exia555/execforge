# Interface Output Contract

A full `design-html` deliverable should include:

- Scope source and non-goals
- Target user and primary job
- Screen inventory
- Information hierarchy for each major screen
- Primary and secondary actions
- State coverage:
  - default
  - loading
  - empty
  - validation
  - error
  - unauthorized
  - success
- Responsive behavior notes
- Accessibility notes:
  - heading structure
  - focus order
  - keyboard interaction
  - status/error announcements
- HTML/CSS delivery notes:
  - landmarks/semantic structure
  - reusable sections/components
  - content dependencies
  - constraints or open issues

If a requested interface cannot be justified by the approved scope, report the contradiction instead of designing around it.
