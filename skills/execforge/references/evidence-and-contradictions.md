# Evidence and Contradictions

## Evidence labels

- **FACT** — Supported by supplied or verified evidence.
- **ASSUMPTION** — Plausible but unconfirmed.
- **INFERENCE** — Reasoned conclusion from facts.
- **UNKNOWN** — Material information is unavailable.

Rules:

1. Confidence is not evidence.
2. Agreement between agents is not proof.
3. Do not invent demand, costs, dates, baselines, regulations, or percentages.
4. A provisional target must be labelled provisional.
5. Material unknowns must appear in the risk register.

## Contradiction types

### Factual

Example: one reviewer says an API exists; the other says only files are supported.

Resolution:

1. Record exact claims and sources.
2. Verify against primary evidence.
3. If unresolved, mark `UNKNOWN`.
4. Prevent the final plan from depending on it.
5. Assign a validation action and owner.

### Strategic

Example: CEO wants real-time; COO recommends batch.

Resolve using:

1. User impact
2. Evidence strength
3. Reversibility
4. Cost
5. Risk
6. Time-to-value
7. Cheapest valid experiment
